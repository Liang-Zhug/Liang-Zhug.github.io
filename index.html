字符驱动框架

1.字符设备驱动介绍

1.1什么是字符设备驱动

字符设备就是一个一个字节，按照字节流进行读写操作的设备，读写数据是分先后顺序的.比如我们最常见的点灯、按键、IIC、SPI，

LCD 等等都是字符设备，这些设备的驱动就叫做字符设备驱动。

1.2Linux应用程序是如何调用驱动程序呢？

当驱动程序加载成功后便会在/dev下生成一个对应文件，应用程序通过对这个名为/dev/xxx的文件进行相应的操作即可实现对硬件的控制。比如现在有个叫做/dev/led 的驱动文件，此文件是 led 灯的驱动文件。比如现在有个叫做/dev/led 的驱动文件，此文件是 led 灯的驱动文件。应用程序使用 open 函数来打开文件/dev/led，使用完成以后使用 close 函数关闭/dev/led 这个文件。open和 close 就是打开和关闭 led 驱动的函数，如果要点亮或关闭 led，那么就使用 write 函数来操作，也就是向此驱动写入数据，这个数据就是要关闭还是要打开 led 的控制参数。如果要获取led 灯的状态，就用 read 函数从驱动中读取相应的状态。

2.字符设备驱动开发步骤

​ 因为控制硬件最本质还是去操作寄存器，在Linux驱动开发过程中肯定也是要初始化相应的外设寄存器。只是在 Linux 驱动开发中我们需要按照其规定的框架来编写驱动，所以说学 Linux 驱动开发重点是学习其驱动框架。在这个框架里面从宏观到微观的说，首先要先将我们编写的驱动加载到内核里面，然后要进行注册字符设备，其目的就是让设备与内核中的数据结构绑定，允许用户空间程序通过标准的文件操作接口（如打开、读、写等）访问该设备，然后就是实现设备的具体操作函数，当应用app调用对应的read，write等函数时，内核会自动去调用驱动中的对应函数。

2.1驱动模块的加载和卸载

​ 要想内核能够使用驱动程序，首先要将驱动加载进内核，Linux里面有两种方式加载驱动，一种是直接将驱动编译进内核里面，这样当内核启动时就会自动的运行驱动程序。第二种是将驱动编译成模块(Linux 下模块扩展名为.ko)，使用iosmod命令即可将模块化的驱动程序加载进内核。模块有加载和卸载两种操作，我们在编写驱动的时候需要注册这两种操作函数，模块的加载和卸载注册函数如下：

module_init 函数用来向 Linux 内核注册一个模块加载函数，参数 xxx_init 就是需要注册的具体函数，当使用insmod命令加载驱动的时候，xxx_init 这个函数就会被调用。module_exit()函数用来向 Linux 内核注册一个模块卸载函数，参数 xxx_exit 就是需要注册的具体函数，当使用rmmod命令卸载具体驱动的时候 xxx_exit 函数就会被调用。而这两个xxx_init和xxx_exit就是驱动的入口和出口函数，通常用__init和__exit来修饰。字符设备驱动模块加载和卸载函数模板如下：

2.2字符设备的注册与注销

​ 当驱动加载成功后需要注册字符设备，同样，卸载驱动模块的时候要注销掉字符设备。字符设备的注册和注销函数原型如下所示:

  • major：主设备号，Linux下的设备都会有一个设备号，设备号分为主设备号和次设备号，主设备号表示某一个具体的驱动，次设备号表示使用这个驱动的各个设备，如一个LED驱动程序下可能有很多个LED灯。主设备号的范围是0~4095，我们在使用设备号的时候要确保此设备号没有驱动程序在用，可以使用cat /proc/devices来查看当前系统已使用的设备号。申请设备号有静态和动态两种方式，静态就是直接设置一个没有使用的设备号的值，例如int major = 300;动态申请是调用alloc_chrdev_region然后系统就会为你自动分配一个没有使用的设备号。
  • name：设备名称
  • fops：结构体 file_operations 类型指针，指向设备的操作函数集合变量

一般字符设备的注册在驱动模块的入口函数 xxx_init 中进行，字符设备的注销在驱动模块的出口函数 xxx_exit 中进行

2.3实现设备的具体操作函数

​ 驱动程序让用户具备操作硬件设备的能力，在Linux里面，设备就是文件，我们可以对它进行打开，读写，关闭等，对于字符设备，Linux提供了一个操作集合file_operations结构体，其原型定义在/include/linux/fs.h下

对于新知识的看法

我是一个非常喜欢学习的人，梦想就是成为一名全栈（前后端，嵌入式软硬件），但其实我连一项都没有做好，我失败的原因有几条，现在总结下来充当教训。

没人指导，自己乱学

  • 没有人指导，导致自己盲目乱学，指导现在，我几乎也都是自学状态，没有人告诉我该怎样做。大一的时候学PCB画电路板，老师上来就一顿讲，每个人都是比较懵的，封装、库都是上B站上搜的才懂什么意思，最后这门课也是不了了之。然后听别人说后端工资高就去学后端，我全然忘记自己在高中是因为电容的串并联才选择的电子信息工程，整个大学期间就是很盲目的状态。

容易被其他技术所吸引

  • 在学习一门技术的时候被另一门技术所吸引，在学电子的时候去学后端，学后端又想着转回嵌入式，总感觉其他行业要好学，一直没有把一门技术学精通。这样我会对每一个行业都知道一些，但却入不了门，在此期间，还花了很多时间学习前端，单片机，汇编。几乎每次学习之后，都不会再用，等到下次在用到的地方却早已忘干净。

没有坚持下去的耐心

  • B站上面每个视频下面对应的都有对应的推荐视频，然后每个视频里面也会有弹幕。这些东西都会对我进行干扰，例如当我看某位博主讲的数据结构，然后推荐视频就会有播放量更高的另一位数据结构教程，弹幕可能也会说不如谁谁谁，我可能就会跳来跳去，到最后花了很久又回到了起点。或者这一段时间打算学习Linux，但是B站推荐页会出现一些要想更加精通Linux，先要学习汇编，自己手写一个简单的操作系统等等，这是我就会去看，跟着博主敲代码，自己一点都不理解，到最后发现这是一个收费视频，最后也感觉坚持不下去，不进行付费，然后也不了了之。我刚开始的目的就是想学一下Linux的应用编程，最后竟然变成精通它，这种急于求成的态度是我之前所忽视的最大阻碍。

总结

  • 由于昨天我刷B站的时候刷到一个stm32的教程，我竟脑子里面想着学习它，但是我仔细想了想我过去的经历，学习它，第一天很有精力配置环境，然后后面越来越摆烂。恰好前天看了<<社交网络>>，于是我给自己总结：想好自己的终极目标是什么，不要为了一切其他事情干扰自己。

2025年主题骑行与编程

骑行与编程

  • 在23年十月份左右，由于我的电瓶车受损，于是便买来一辆山地车进行代步，这还是我人生中的第一辆变速车，第一次骑的时候便觉得非常轻，和我平常骑的共享单车有很大不同，于是当天便骑了20KM，但却丝毫不觉得累。于是，每天我就要起上20-30KM，在骑行的过程中，能够忘掉一切压力，忘掉秋招，忘掉学习，在骑行中逃避自己。2025年，我打算换一辆国产vook one公路车，一是因为公路车比山地车能轻个5,6KG，骑行感受会更好、二是公路车可以骑的更远，爬坡更加轻松，能更好的锻炼身体。
  • 现在我要每天进行编程，之前学习的时候，每次看完一个知识点，就感觉自己会了，眼高手低，导致再过两天根本看不懂，孰能生巧，偷懒骗的只有自己，现在的我已经深刻的感受到了。2025的我没有太大的追求，坚持骑行锻炼身体，坚持编程找到好工作，加油！希望365天后的自己会感谢现在的自己。

第一篇博客

第一篇博客

一.为什么要创建个人博客文章

  • 这两天在学习QT网络编程和多线程时，无意间看到了爱编程的大丙的文章，觉得写的非常好，很有条理，之后又在他的网站上看了其他文章，便觉得记录，分享应该是一种很美好的事情。大学期间学过很多知识，也做过很多笔记，可是每次写完都不在看了，条理很不清晰，笔记放的地方也比较杂乱，如果在博客里面分好标签应该更容易复习一下。之前也在CSDN上写过几篇，但是却总是担心自己都没理顺，能让别人看懂吗？网上那么多同类优质博客，写的不如别人还有必要写吗？总是在担心，却直到今天才有下定决心开始写博客，反正先开始吧，如果写的东西能够帮助一个人，那就有它的意义。

二.现在的处境

  • 现在是2024年12月31日22点32分，离2025年还有不到两小时，现在的我很是迷茫，秋招没有毅力坚持投简历，选择了摆烂，直到现在还没找到工作。体重连年攀升，现在已经飙到92KG。离2025的倒计时就像一个炸弹的引线，随时能唤起我的焦虑。大学期间，我总觉得比同班人大多数要懂得多，可是，离开了我们这个二本，才知道自己是多么的菜，这种反差感让我对找工作有种恐惧感，觉得自己什么都不能胜任，于是一直在逃避，近几个月，每天早上到达出租屋后，便开始发呆，浪费了很多时间，但是现在我已经和自己和解，放平心态，不再焦虑。

三.2025年的展望

  • 2025年的首要愿望就是在3月份之前找到工作，其次能够顺利从学校毕业（本科期间挂科较多），然后能够培养几个兴趣爱好（骑行），最后，每天都要保证代码量。
